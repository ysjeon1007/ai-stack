이 프로젝트를 아래 주어진 서비스에 대한 랜딩페이지로서 구성하라.

먼저 컬러 시스템을 설정하라. 이 서비스의 accent 컬러를 '토스' 서비스와 유사한 파란 컬러로 삼을 것이다.
60-30-10 규칙을 따르도록 팔레트를 설정하라.

그리고 디자인 규칙들(@.ruler/design.md)에 맞게 랜딩페이지를 구현하면 된다. 아래 프로세스를 참고하라.

- 타겟 유저 선택
- 섹션 구성하기
  - 무조건 포함
    - Header
    - Footer
    - Hero
    - Why XXX?
    - CTA
  - 이외에도 이 서비스를 고유하게 설명하는 특별한 섹션 2~3개 추가
- 섹션 모두 구현하기

- 이미 구현된 섹션 컴포넌트들을 활용하되, 문구/배치/shadow 등을 해당 서비스의 컨셉에 맞게 조금씩 수정하라.
- 이미지가 필요한 경우 nanobanana mcp를 사용해 생성한 뒤 경로 설정하라. 디자인 가이드를 따르도록, 미니멀한 일러스트로서 생성되도록 주의해서 프롬프트를 입력하라.
- 로고 이미지도 적절히 생성해서 설정하라. ai saas에 걸맞는 미니멀한 심볼로, 서비스 특성을 드러낼 수 있게 설정하라.
- 섹션에 포함되는 카피는 짧고 간결하지만 핵심을 잘 파악할 수 있게, 전문가로서 작성해야한다.

사이트 메타데이터는 @src/config/site-data.json 를 옳은 값으로 수정하면 된다.

lint, build 에러가 없음을 보장하라.

## 1. 서비스 소개

### 1-1. 기본 개념

- **브랜드명**: Vooster (vooster.ai)
- **포지션**: “AI Product Manager / PM Agent for AI Coding”

  - 공식 사이트에서 “The PM Agent for AI Coding”, “Your AI Partner for Project Management”라고 정의하고 있음.([vooster.ai][1])

- **코어 가치**

  - 아이디어만 입력하면 **PRD → TRD → 태스크 → 코드 템플릿**까지 자동 생성하여, 개발자가 바로 “바이브코딩(vibe coding)”을 시작할 수 있게 만드는 도구.([gpters.org][2])
  - Cursor 같은 AI 코딩 IDE와 **MCP(Multi-Command/Client Protocol)**로 연동하여, 기획·설계 문서와 실제 개발 작업(코딩, 이슈 처리)을 연결하는 “AI PM 허브” 역할을 함.([메일리][3])

### 1-2. 어떤 문제를 풀려고 하나?

- 실무에서 **기획–문서화–태스크 분해–개발 연결**이 굉장히 번거롭고,

  - PRD/TRD 포맷을 아는 사람도 적고,
  - 아이디어는 많지만 체계적으로 정리·쪼개서 개발까지 이어 가는 능력은 부족한 경우가 많음.

- Vooster는 이 과정을 **질문 기반 템플릿 + AI 자동화**로 단축:

  - “아이디어 1줄 설명”, “타깃 유저”, “비슷한 서비스”, “MVP 범위”, “기능 Must/ Nice to have” 등 핵심 질문을 던지고([gpters.org][2])
  - 답변을 기반으로 PRD·TRD·태스크를 자동 생성 → Cursor와 연동해 실코드까지 이어지는 구조.

## 2. 주요 기능

### 2-1. Detailed PRD Writing (PRD 자동화)

공식 사이트와 사용후기를 종합하면, PRD 관련 기능은 다음과 같습니다.

- **질문 기반 PRD 템플릿 제공**

  - “프로젝트 아이디어 한 줄 설명, 타깃 유저, 유사 서비스, 해결하는 베네핏, MVP 범위, Must/Nice 기능” 등 일련의 질문 리스트를 제공.([gpters.org][2])

- **최적화된 프롬프트 내장**

  - 사용자가 위 질문에 답만 하면, Vooster가 내부적으로 AI 프롬프트를 조합해 **PRD 초안 문서**를 자동 생성.([vooster.ai][1])

- **문서화 메뉴 & 다운로드**

  - “설계 문서화” 메뉴에서 PRD, TRD, Task 문서를 확인하고, 다운로드할 수 있는 UI를 제공한다고 설명.([gpters.org][4])

**정리**: PRD를 처음 써보는 기획자·디자이너·개발자도 질문에만 답하면 “실무에서 쓸 수 있는 수준의 PRD 초안”을 빠르게 확보할 수 있도록 돕는 기능.

---

### 2-2. TRD 자동 생성 (Technical Requirement Doc)

- PRD를 기반으로 **TRD 초안**을 버튼 한 번에 생성.([gpters.org][2])
- TRD에는 다음과 같은 요소들이 포함되는 것으로 소개됨:

  - 기능 단위 API 목록 및 데이터 흐름
  - 필요한 UI 설명
  - 구현 단계별 로드맵 등([gpters.org][5])

- 생성된 TRD는(사용 후기 기준) 다시 GPT/Claude 같은 모델들에게 넘겨 **리뷰·수정·보완**하는 식으로 쓰이고 있음.([gpters.org][2])

**정리**: Vooster는 “기획 문서” 수준이 아니라, 실제 개발자가 읽고 바로 구현할 수 있는 수준의 **기술 명세서**까지 자동화하는 것이 특징.

---

### 2-3. 태스크 분해 & 프로젝트 계획

- PRD/TRD를 기반으로 **개발 태스크를 자동 분해**:

  - 사용자 질문에 답하면서 태스크를 생성하고, 우선순위를 붙이는 형태.([gpters.org][2])

- 태스크는 Vooster 대시보드에서 관리 가능:

  - “MVP 전체 자료를 모아 둔 대시보드”에서 PRD, TRD, Task를 한 곳에서 관리하는 구조로 소개됨.([gpters.org][4])

- 로드맵/마일스톤 관리 기능은 현재 **로드맵 상의 예정 기능**으로도 별도 표기되어 있음.([vooster.ai][1])

---

### 2-4. Structured Codebase (코드 템플릿 & 바이브코딩 연계)

- 공식 사이트에서는 “Start vibe coding with pro-created code templates”라고 명시.([vooster.ai][1])
- 실제 사용 사례에서는:

  - Vooster가 생성한 TRD/태스크를 Cursor에 붙여넣어 구현을 진행하거나,
  - 이메일 템플릿 자동화, 랜딩페이지 제작, 개인 블로그 제작 등에서 **React/Next.js 기반 코드 구조**를 AI에게 일괄 지정하는 용도로 활용.([gpters.org][6])

- 별도 생태계로 **@vooster/cli**라는 패키지가 있어 Cursor와 같은 AI 코딩 에이전트와 프로젝트 룰을 관리하는 CLI 툴로 소개됨 (npm 검색 결과).([Npm][7])

**정리**: 문서 생성에서 멈추지 않고, **실제 코드 생성 단계에 필요한 구조/템플릿까지 제공**해 “아이디어 → 코드”의 거리를 크게 줄이는 기능을 지향.

---

### 2-5. Cursor & MCP 연동

- 여러 아티클과 인터뷰에서 **Vooster를 MCP로 Cursor에 연결하는 시나리오**가 강조됨.([메일리][3])

  - Vooster가 만든 PRD와 태스크를 Cursor의 작업 맥락으로 넣어, AI 코딩 에이전트가 스스로 할 일 관리와 구현을 진행하는 식.

- 목표는 “기획 ↔ 코딩 ↔ 배포”를 하나의 자동화된 워크플로로 묶는 것.([gpters.org][6])

---

### 2-6. 로드맵 상의 향후 기능

공식 사이트의 Roadmap 섹션에 적힌 예정 기능들:([vooster.ai][1])

- **Context Integration Management**

  - 회사/서비스의 다양한 데이터를 이해하고, 각 태스크에 정확한 컨텍스트를 제공하는 기능.

- **Team Collaboration Features**

  - Cursor를 도입한 개발팀을 위한 협업 기능 (티켓 처리 전체를 MCP 기반으로 수행).

- **Milestone & Roadmap Management**

  - 티켓 처리 데이터를 분석해 진행 상황과 성과를 한 눈에 보여주는 기능.

- **Insight Generation**

  - GA 및 기타 서비스 데이터를 분석해 인사이트와 액션 플랜을 제안.

## 3. 사용법 (일반적인 워크플로)

아래는 공식 사이트 및 사용 후기들을 종합해 정리한, **실제 사용 흐름 예시**입니다.

> _주의: 세부 UI는 서비스 업데이트에 따라 달라질 수 있고, 일부는 사례 기반 추론입니다._

### 3-1. 프로젝트 생성

1. **vooster.ai 접속 후 회원가입/로그인**
2. “프로젝트 생성” 혹은 유사 버튼을 눌러 새 프로젝트를 만들고,
3. 프로젝트 이름·간단한 아이디어 설명 등을 입력.

※ 무료 플랜 기준:

- 월 100 크레딧, 최대 3 프로젝트 제한, 기본 AI 모델 사용.([vooster.ai][1])

---

### 3-2. PRD 템플릿 질문에 답변하기

1. PRD 템플릿 화면에서 다음과 같은 질문에 답:

   - 이 서비스 아이디어를 한 줄로 설명해보라
   - 어떤 유저가 어떤 문제를 겪고 있는가
   - 유사 서비스와 차별점
   - 예상 유저/타깃
   - 반드시 필요한 기능 vs 있으면 좋은 기능([gpters.org][2])

2. 이 답변은 Vooster 내부의 프롬프트에 들어가 PRD 초안으로 재구성됨.
3. 생성된 PRD를 에디터에서 검토/수정하고, 필요하면 다운로드.

---

### 3-3. TRD 생성 및 보완

1. “TRD 생성” 버튼(또는 유사 액션)을 눌러 PRD 기반 TRD 초안을 생성.([gpters.org][2])
2. 생성된 TRD에는 기능별 API, 데이터 흐름, 필요한 UI 요소, 구현 단계 등이 포함.
3. TRD 내용을 다시 GPT/Claude 등으로 가져가 세부 보완을 하고, 수정된 버전을 다시 프로젝트에 반영.

---

### 3-4. 태스크 분해 & 대시보드 관리

1. Vooster에서 제공하는 “태스크 생성/분해” 기능으로 TRD를 기반으로 한 구체적 태스크 리스트를 생성.([gpters.org][2])
2. 대시보드에서:

   - 전체 MVP 관련 문서(PRD, TRD, Task)를 한 번에 확인하고,
   - 태스크별 상태/우선순위를 관리.([gpters.org][4])

---

### 3-5. Cursor + MCP 연동 후 개발

1. Cursor에서 MCP를 통해 Vooster 프로젝트에 연결.
2. Vooster가 관리하는 PRD/태스크 정보를 Cursor의 컨텍스트로 불러와:

   - “이 태스크 구현해줘”,
   - “이 TRD의 이 섹션을 코드로 옮겨줘” 등의 요청을 자동화.([gpters.org][6])

3. 이메일 템플릿 자동화, 랜딩페이지 제작, 블로그 구축 등 다양한 실험에서
   “기획 ↔ 코딩 ↔ 배포” 전체 과정을 Vooster + Cursor 조합으로 수행하는 사례가 보고됨.([gpters.org][6])

## 4. 차별성

### 4-1. 일반 AI 문서/노션 템플릿과의 차이

1. **PM/개발 특화 도메인**

   - 단순 문장 생성이 아니라 **PRD/TRD/태스크/로드맵**이라는 소프트웨어 제품 개발 도메인에 특화.([gpters.org][2])

2. **질문 기반 구조화**

   - “좋은 질문 리스트”를 기반으로, 초보자도 기획 구조를 따라가게 설계됨 (PRD, TRD, Task 템플릿).([gpters.org][4])

3. **코드 레벨까지 이어지는 설계**

   - TRD 및 코드 템플릿, 그리고 Cursor 연동을 통해 단순 문서 생성에 그치지 않고 “실제 코드 생산” 단계까지 연결.([vooster.ai][1])

### 4-2. AI 코딩 IDE와의 관계

- Cursor, Taskmaster 같은 AI 코딩 도구는 “코드를 잘 짜는 도구”에 가깝다면,
- Vooster는 그 앞단인 **“무엇을 만들 것인가를 정의하고 쪼개는 PM 계층”**에 집중.
- MCP 연동을 통해 “기획/설계에서 정의된 룰·태스크”를 AI 코딩 에이전트에게 그대로 넘긴다는 점이 강점으로 언급됨.([메일리][3])

### 4-3. 한국 바이브코딩 생태계 중심

- 국내 AI 스터디/커뮤니티(GPters 등)를 기반으로 성장하면서,

  - PRD·TRD 개념에 익숙하지 않은 **한국 개발자·디자이너·창업자**를 주요 사용자로 두고 있음.([gpters.org][2])

- 한국어 사례, 튜토리얼, 스터디 커리큘럼에 깊이 녹아 있어,

  - “실제 MVP를 4주 안에 출시” 같은 실전형 교육과 함께 사용되는 툴로 자리 잡는 중.([gpters.org][2])

### 4-4. 생태계 & 커뮤니티 드리븐

- 공식 사이트에서 Discord 커뮤니티를 전면에 내세우며, 유저 피드백 기반으로 빠르게 제품을 진화시키겠다고 명시.([vooster.ai][1])
- 창업자(커서맛피아)는 Cursor 교육, 바이브코딩 챌린지, AI 창업 패키지 등 여러 교육 프로그램/콘텐츠를 운영하며
  Vooster를 중심으로 하는 “AI 솔로프리너/바이브코더 생태계”를 키우는 중.([메일리][3])

## 5. 타겟 고객

### 5-1. 공식 타겟

공식 사이트의 “Perfect for These People” 섹션 기준:([vooster.ai][1])

- **Idea-Driven Creative VibeCoder**

  - 아이디어는 많지만, 디버깅·프롬프트·기획 구조화에 어려움을 느끼는 개발자.

- **AI Friendly Developer**

  - AI와 함께 빠르게 프로토타입을 만들고 싶은 개발자.

- **Startup Team**

  - 작은 팀 규모에서 **PM/기획 인력이 부족**한 스타트업 개발팀.

### 5-2. 실제 사례에서 드러난 타겟

리뷰/블로그 글들을 기반으로 보면:([gpters.org][2])

1. **디자이너 출신 창업자/기획자**

   - UX/UI는 익숙하지만 PRD, TRD는 처음인 사람들 → 질문 기반 템플릿을 통해 빠르게 실전 문서를 뽑아냄.

2. **솔로 개발자/1인 창업자**

   - 혼자 기획·개발·배포까지 해야 하는 사람들 → Vooster로 기획/태스크를 정리하고 Cursor로 구현.

3. **AI 활용에 적극적인 주니어/미드 레벨 개발자**

   - PM 경험은 적지만, AI 도구를 적극 활용해 “실전 프로젝트 경험”을 쌓고 싶은 개발자.

4. **AI 교육/부트캠프 수강생**

   - AI 창업 패키지, 바이브코딩 스터디 등 커리큘럼 내에서
     “아이디어 발굴 → Vooster로 기획/설계 → Cursor로 MVP 구현” 플로우를 실습용으로 사용.

## 6. 요약 & 해석

- **Vooster AI는 단순한 ‘문서 생성기’가 아니라, PRD–TRD–태스크–코드베이스까지 잇는 “AI PM 플랫폼”**입니다.
- 강점은:

  1. PRD/TRD/태스크 자동화라는 **PM 특화 AI**
  2. Cursor·MCP 연동을 통한 **바이브코딩 친화 워크플로**
  3. 한국어 기반의 교육/커뮤니티와 밀접하게 연결된 **실전 사례 풍부함**

- 따라서,

  - “아이디어는 많은데, 기획 문서·태스크 분해·개발 연결까지 혼자 다 하기 벅찬”
    **솔로 개발자, 초기 스타트업 팀, 디자이너/기획자 출신 창업자**에게 특히 적합한 서비스로 보입니다.
